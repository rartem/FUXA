# История изменений

## Следующий релиз

### Добавлено
- **Загрузка и управление пользовательскими шрифтами** - Добавлена возможность загрузки, управления и применения пользовательских шрифтов TrueType (.ttf) в HMI-представлениях для большей гибкости оформления. Включает новый `ResourceType.fonts` в модели ресурсов, сервис `FontLoaderService` для динамической загрузки шрифтов из директории ресурсов, внедрения стилей `@font-face` и управления реестром шрифтов в `Define.fonts`, а также компонент `LibFontsComponent` с интерфейсом загрузки .ttf файлов.

- **Комплексный White-Labeling для брендирования приложения** - Реализованы настройки для кастомизации визуальной идентичности приложения. Пользователи могут устанавливать пользовательское название приложения, логотип и favicon, а также скрывать подпись "powered by". Эти настройки применяются к заставке, шапке, боковой навигации, экрану входа, информационному диалогу, AR-представлению, атрибуции карт и заголовкам PDF-отчётов.

- **Логирование событий и представление истории** - Реализована новая система логирования и просмотра событий приложения для улучшения аудируемости и операционной прозрачности. Включает новый `EventsService` и `EventsHistoryComponent` на клиенте для получения, логирования и отображения событий с фильтрацией, сортировкой и пагинацией. Новые API-эндпоинты (`/api/events`, `/api/events/log`, `/api/events/clear`) для управления данными событий. Выделенный `EventsManager` на сервере с SQLite-хранилищем для сохранения записей событий. Автоматическое логирование ключевых действий: вход/выход пользователей, подключение/отключение клиентов, взаимодействие пользователей с датчиками HMI (клики, ввод значений), выполнение скриптов (API-триггеры, по расписанию или интервальным). Интеграция представления истории событий в маршрутизацию приложения и меню настройки.

- **Иерархическая организация папок для HMI-представлений** - Добавлена возможность группировки HMI-представлений в папки, что существенно улучшает организацию проекта и навигацию в редакторе. Включает новую модель `ViewFolder` и её интеграцию в структуру проекта `Hmi`. Расширение `ProjectDataCmdType` и `ProjectService` операциями CRUD для папок. Переработанный `EditorViewsListComponent` для отображения навигируемой древовидной структуры представлений и папок. Добавлены опции UI для создания, переименования, удаления и перемещения представлений и папок. Обновлены механизмы загрузки и сохранения проекта на бэкенде для персистентности данных папок.
